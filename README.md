# Course Completion Prediction

## Описание проекта

Данный проект посвящён прогнозированию завершения онлайн-курса студентом на основе ранних поведенческих данных.  

Проект демонстрирует полный цикл работы с табличными данными:  
* сбор и объединение данных из нескольких источников
* очистку и предобработку
* feature engineering с поведенческой интерпретацией
* корректную работу с несбалансированными классами
* моделирование и анализ результатов
* интерпретацию полученных выводов

## Цель проекта

Построить интерпретируемую модель машинного обучения для прогнозирования вероятности успешного завершения курса на раннем этапе обучения.  

Задача сформулирована как бинарная классификация:  
* 1 — студент завершил курс  
* 0 — студент не завершил курс  

Особое внимание уделено анализу precision–recall trade-off и подбору оптимального порога классификации.  

## Используемые данные

В проекте использованы два исходных датасета, образовательной платформы:  

**events_train.csv**  
Данные о действиях студентов при взаимодействии со степами курса:  
* step_id — идентификатор шага  
* user_id — анонимизированный идентификатор пользователя  
* timestamp — время события в формате Unix time  
* action — тип события:  
  * discovered — переход к шагу  
  * viewed — просмотр шага  
  * started_attempt — начало попытки решения  
  * passed — успешное выполнение практического шага  

**submissions_train.csv**   
Данные об отправке решений к практическим заданиям:  
* step_id — идентификатор шага
* timestamp — время отправки решения в формате Unix time
* submission_status — статус проверки решения
* user_id — анонимизированный идентификатор пользователя

На основе этих таблиц были сформированы агрегированные поведенческие признаки на уровне пользователя.

Целевая переменная отражает факт завершения курса:
* 1 — студент завершил курс
* 0 — студент не завершил курс

В данных наблюдается дисбаланс классов.

## Feature Engineering

На основе исходных таблиц были сконструированы агрегированные поведенческие признаки, отражающие раннюю динамику обучения:
* количество выполненных заданий
* интенсивность активности в первые дни
* частота взаимодействий
* интервалы между действиями
* производные метрики вовлечённости

Признаки имеют логическую интерпретацию и направлены на выявление поведенческих паттернов, связанных с успешным завершением курса.

## Анализ корреляций

Проведён анализ корреляционной структуры признаков, который показал:
* отсутствие сильной мультиколлинеарности
* наличие ожидаемых зависимостей между поведенческими метриками
* различную степень влияния признаков на целевую переменную

Использование логистической регрессии с L2-регуляризацией позволило дополнительно стабилизировать оценки коэффициентов.

## Модели

В проекте использованы:
* Logistic Regression (L2-регуляризация, solver="liblinear")
* Random Forest

Для корректной оценки моделей применялся stratified train/test split с сохранением пропорций классов.

Дополнительно проведён анализ влияния порога классификации на качество предсказаний.

## Результаты

Логистическая регрессия и Random Forest продемонстрировали сопоставимое качество классификации.

Изменение порога классификации позволило варьировать баланс между precision и recall для класса завершивших курс.

Анализ показал:
* устойчивость моделей
* чувствительность качества к выбору порога
* влияние поведенческих признаков на вероятность завершения курса
